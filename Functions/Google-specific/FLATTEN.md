# FLATTEN

## Description

The **FLATTEN function** is a Google Sheets-exclusive function that collapses a two-dimensional array into a single column by placing each row's values sequentially beneath the previous row's values. This powerful array manipulation function transforms multi-column data into a single column, reading values left-to-right across each row before moving to the next row. FLATTEN is essential for converting wide data formats into tall formats, preparing data for analysis functions that require single-column input, and transforming grid layouts into linear sequences.

The function processes its input by iterating through the array row by row, taking each cell value from left to right and placing it in a vertical stack. For example, a 3x3 array becomes a 9x1 column, with the first row's three values on top, followed by the second row's three values, then the third row's. Empty cells are preserved in the output, maintaining the positional relationship of data. FLATTEN works with any range or array, including those generated by other functions like SPLIT, making it invaluable for complex data transformation pipelines.

One of the most powerful uses of FLATTEN is combining it with QUERY, UNIQUE, or aggregation functions that need single-column input. For instance, counting unique values across multiple columns becomes trivial with COUNTUNIQUE(FLATTEN(A:C)). Similarly, finding the maximum value across an entire grid uses MAX(FLATTEN(range)). FLATTEN also enables advanced array manipulations when combined with FILTER and SORT, allowing operations that would otherwise require complex nested formulas or Apps Script.

FLATTEN is exclusively available in Google Sheets and has no direct equivalent in Microsoft Excel. Excel users must use alternative approaches such as Power Query's unpivot feature, VBA macros, or helper column formulas to achieve similar results. Excel 365's dynamic array functions like TOCOL() and TOROW() provide some similar capabilities but were introduced later and have different syntax. For teams requiring cross-platform compatibility, FLATTEN formulas will need to be converted to alternative methods when sharing with Excel users.

## Syntax

> [!f(x)] FLATTEN Syntax
>
> ```
> =FLATTEN(range1, [range2, ...])
> ```

### Parameters

| Parameter | Required | Description |
|-----------|----------|-------------|
| `range1` | Yes | The first range or array to flatten. Can be a cell reference (A1:C10), an array constant ({1,2,3;4,5,6}), or an array returned by another function. The values are read row by row, left to right. |
| `range2, ...` | No | Additional ranges or arrays to append to the flattened result. Each additional range is flattened and appended below the previous range's flattened values. |

### Return Value

Returns a single-column array containing all values from the input range(s). Values are ordered by reading each row left-to-right, then proceeding to the next row. Empty cells in the input become empty cells in the output. The output array's height equals the total number of cells in all input ranges.

## Examples

> [!f(x)] FLATTEN Examples

### Example 1: Basic 2x3 Array Flatten
```
=FLATTEN(A1:C2)
```
**Scenario:** A1:C2 contains: [1,2,3] in row 1, [4,5,6] in row 2
**Result:** A single column: 1, 2, 3, 4, 5, 6

**Explanation:** FLATTEN reads the first row (1,2,3) left to right, then the second row (4,5,6), stacking all values in a single column.

---

### Example 2: Flatten Array Constant
```
=FLATTEN({1,2,3;4,5,6;7,8,9})
```
**Scenario:** Flattening a 3x3 array constant
**Result:** A single column: 1, 2, 3, 4, 5, 6, 7, 8, 9

**Explanation:** Array constants use semicolons for row separators and commas for column separators. FLATTEN processes them the same as cell ranges.

---

### Example 3: Flatten Multiple Ranges
```
=FLATTEN(A1:B2, D1:E2)
```
**Scenario:** Two separate 2x2 ranges
**Result:** 8 values in a single column—first range's values, then second range's values

**Explanation:** Multiple ranges are processed sequentially. All values from the first range appear before any values from the second range.

---

### Example 4: Count Unique Across Multiple Columns
```
=COUNTUNIQUE(FLATTEN(A:C))
```
**Scenario:** Counting distinct values appearing anywhere in columns A, B, or C
**Result:** A single number representing unique value count

**Explanation:** FLATTEN converts the multi-column data into a single column, which COUNTUNIQUE can then process. This is one of the most common FLATTEN patterns.

---

### Example 5: Find Maximum Across a Grid
```
=MAX(FLATTEN(A1:E10))
```
**Scenario:** Finding the highest value in a 5x10 grid of numbers
**Result:** The maximum value from all 50 cells

**Explanation:** While MAX can work on a range directly, this pattern demonstrates how FLATTEN enables any single-column function to work with multi-column data.

---

### Example 6: Remove Empty Cells After Flattening
```
=FILTER(FLATTEN(A1:C10), FLATTEN(A1:C10)<>"")
```
**Scenario:** Flatten a range and exclude empty cells
**Result:** Only non-empty values from the flattened array

**Explanation:** FILTER works on arrays, so FLATTEN must be applied to both the data and the condition. This removes gaps from sparse data.

---

### Example 7: Flatten SPLIT Results
```
=FLATTEN(SPLIT(A1, ","))
```
**Scenario:** A1 contains "a,b,c,d,e" and you want a vertical list
**Result:** Each letter in its own row: a, b, c, d, e

**Explanation:** SPLIT produces a horizontal array; FLATTEN converts it to vertical. Useful for processing comma-separated values into rows.

---

### Example 8: Combine and Deduplicate Multiple Columns
```
=UNIQUE(FLATTEN(A:C))
```
**Scenario:** Get unique values that appear anywhere in columns A, B, or C
**Result:** A single column of distinct values

**Explanation:** FLATTEN combines all columns, then UNIQUE removes duplicates. This creates a master list of all unique values across multiple columns.

---

### Example 9: Sum Conditional Across Multiple Columns
```
=SUMPRODUCT((FLATTEN(A1:C10)>0)*(FLATTEN(A1:C10)))
```
**Scenario:** Sum only positive values from a grid
**Result:** Total of all positive values

**Explanation:** FLATTEN enables SUMPRODUCT-style conditional aggregation across multi-column ranges. The condition and values are both flattened.

---

### Example 10: Flatten for QUERY Source
```
=QUERY(FLATTEN(A1:D100), "SELECT Col1 WHERE Col1 IS NOT NULL")
```
**Scenario:** Query a flattened range to filter values
**Result:** All non-null values from the original grid

**Explanation:** QUERY works with flattened data, enabling SQL-like filtering and aggregation across what was multi-column data.

---

### Example 11: Pair Values from Two Columns
```
=ARRAYFORMULA(INDEX(FLATTEN(A2:A),SEQUENCE(ROWS(A2:A)*2-1,1,1,0.5)+(MOD(SEQUENCE(ROWS(A2:A)*2,1,0),2)*(ROWS(A2:A)-0.5))))
```
**Scenario:** Interleave values from multiple columns (advanced)
**Result:** Complex interleaved pattern

**Explanation:** While complex, this demonstrates FLATTEN's use in advanced array manipulations for reshaping data.

---

### Example 12: Create Union of Multiple Lists
```
=SORT(UNIQUE(FLATTEN(A2:A, B2:B, C2:C)))
```
**Scenario:** Combine three lists into one sorted, deduplicated list
**Result:** Alphabetically sorted list of all unique values from three columns

**Explanation:** FLATTEN merges the columns, UNIQUE removes duplicates, SORT orders the result. This creates a clean, consolidated list.

## Common Errors

| Error | Cause | Solution |
|-------|-------|----------|
| `#VALUE!` | Invalid range reference | Verify all ranges are valid cell references or arrays |
| `#REF!` | Output would overflow available space | Ensure sufficient rows below the formula for the flattened result |
| `Unexpected order of values` | Misunderstanding row-by-row processing | Remember FLATTEN reads left-to-right across each row, then moves down |
| `Too many empty cells in output` | Source range has many blanks | Use FILTER(FLATTEN(...), FLATTEN(...)<>"") to remove empties |
| `Function not recognized` | Using in Excel | FLATTEN is Sheets-only; use Excel's TOCOL or alternative methods |
| `Circular reference` | FLATTEN range includes the formula cell | Ensure the formula is outside the range being flattened |
| `#NUM! with very large ranges` | Memory or calculation limits | Break large operations into smaller chunks |
| `Values not in expected order` | Multiple ranges processed | Ranges are processed in order given; first range's values come first |

## Use Cases

### [[Consolidating Survey Responses Across Multiple Columns]]

**Scenario:** A survey allows respondents to list up to 5 favorite products in separate columns (Product1, Product2, Product3, Product4, Product5). The analysis team needs to count which products are mentioned most frequently across all columns and create a ranking of product popularity.

**Implementation:**
```
Survey data structure:
Column A: Respondent ID
Columns B-F: Product1 through Product5 (text)

Flatten all product mentions:
=FLATTEN(B2:F100)

Count each product's total mentions:
=QUERY(FLATTEN(B2:F100), "SELECT Col1, COUNT(Col1) WHERE Col1 <> '' GROUP BY Col1 ORDER BY COUNT(Col1) DESC LABEL COUNT(Col1) 'Mentions'")

Get top 10 products:
=QUERY(FLATTEN(B2:F100), "SELECT Col1, COUNT(Col1) WHERE Col1 <> '' GROUP BY Col1 ORDER BY COUNT(Col1) DESC LIMIT 10")

Count unique products mentioned:
=COUNTUNIQUE(FLATTEN(B2:F100))

Find products mentioned only once:
=UNIQUE(FILTER(FLATTEN(B2:F100), COUNTIF(FLATTEN(B2:F100), FLATTEN(B2:F100))=1))
```

**Business Application:** Multi-response survey questions create wide data that's difficult to analyze with standard functions. FLATTEN enables treating all responses as a single pool for counting, ranking, and analysis. Marketing teams can quickly identify top products without manually combining columns or writing complex formulas.

**Technical Details:** QUERY with GROUP BY and COUNT provides frequency analysis after flattening. The COUNTIF pattern finds items appearing exactly N times. FILTER removes blank responses (common when respondents don't fill all slots).

### [[Cross-Column Duplicate Detection]]

**Scenario:** A data quality team needs to identify duplicate values that appear across multiple columns in a dataset. For example, finding phone numbers that appear in both "Primary Phone" and "Secondary Phone" columns, or identifying products listed in multiple category columns.

**Implementation:**
```
Duplicate detection structure:
Columns A-D: Various data columns that might contain duplicates

Find all values appearing more than once (across all columns):
=UNIQUE(FILTER(FLATTEN(A:D), COUNTIF(FLATTEN(A:D), FLATTEN(A:D))>1))

Count how many times each value appears:
=QUERY(FLATTEN(A2:D100), "SELECT Col1, COUNT(Col1) WHERE Col1 <> '' GROUP BY Col1 HAVING COUNT(Col1) > 1")

Mark rows containing cross-column duplicates:
=IFERROR(IF(COUNTIF(FLATTEN($B$2:$E$100), A2)>1, "Duplicate", ""), "")

Create duplicate report with locations:
Combine FLATTEN with REGEXMATCH and row tracking for detailed reports
```

**Business Application:** Data deduplication often focuses on single-column duplicates, but cross-column duplicates can be equally problematic. Customer databases might have the same phone number in different fields across records. FLATTEN enables holistic duplicate detection across the entire dataset.

**Technical Details:** The COUNTIF on flattened data counts occurrences across all columns. Values appearing more than once are duplicates. For large datasets, consider breaking into smaller ranges or using Apps Script for performance.

### [[Multi-Column Pivot for Reporting]]

**Scenario:** A finance team has budget data organized with months as columns (Jan, Feb, Mar, etc.) and needs to transform it into a tall format for import into a reporting tool. Each row should contain one record with the category, month, and value.

**Implementation:**
```
Source data structure:
Column A: Category
Columns B-M: Jan through Dec values

Create month labels for each value:
=FLATTEN(ARRAYFORMULA(REPT(B1:M1 & CHAR(10), ROWS(A2:A100))))

Create category labels for each value:
=FLATTEN(TRANSPOSE(ARRAYFORMULA(REPT(A2:A100 & CHAR(10), 12))))

Flatten the values:
=FLATTEN(B2:M100)

Combined unpivot approach (simplified):
For each row, create 12 rows (one per month) with:
- Original row's category
- Month name
- That month's value

Full unpivot formula (complex):
=ARRAYFORMULA({
  FLATTEN(TRANSPOSE(ARRAYFORMULA(REPT(A2:A & CHAR(10), 12)))),
  FLATTEN(ARRAYFORMULA(REPT(B1:M1 & CHAR(10), ROWS(A2:A)))),
  FLATTEN(B2:M)
})
```

**Business Application:** Reporting tools and databases often require "tall" or "normalized" data formats rather than "wide" spreadsheet layouts. FLATTEN is the core component of unpivoting operations that transform spreadsheet-friendly layouts into database-friendly formats.

**Technical Details:** The unpivot operation requires creating repeated labels that align with flattened values. REPT creates repeated text, TRANSPOSE adjusts orientation, and ARRAYFORMULA enables array operations. This is a complex transformation that may be easier with Apps Script for large datasets.

### [[Validating Data Across Multiple Entry Points]]

**Scenario:** A system has multiple data entry forms that feed different columns, and administrators need to validate that all entries conform to a master list of allowed values. Any value in any column must exist in the approved list.

**Implementation:**
```
Validation structure:
Columns B-F: Data entry columns from various forms
Column Z: Master list of approved values

Find all invalid entries:
=FILTER(FLATTEN(B:F), ISNA(MATCH(FLATTEN(B:F), $Z:$Z, 0)))

Count invalid entries:
=SUMPRODUCT(--ISNA(MATCH(FLATTEN(B2:F100), $Z$2:$Z$500, 0)))

Mark columns with invalid data:
=IF(SUMPRODUCT(--ISNA(MATCH(FLATTEN(B2:B100), $Z:$Z, 0)))>0, "Invalid entries", "Valid")

Create validation report:
=QUERY({FLATTEN(B2:F100), ARRAYFORMULA(IF(ISNA(MATCH(FLATTEN(B2:F100), $Z:$Z, 0)), "Invalid", "Valid"))},
  "SELECT Col1, Col2 WHERE Col2 = 'Invalid'")
```

**Business Application:** Data validation across multiple columns is challenging without FLATTEN. When values can appear in any of several columns, validating against a master list requires checking all columns simultaneously. FLATTEN enables this single-pass validation approach.

**Technical Details:** MATCH returns #N/A for values not in the master list. ISNA identifies these failures. FILTER extracts only the failures. The QUERY variant creates a detailed report of all invalid values.

## Platform Differences

FLATTEN is **exclusively available in Google Sheets** and has no direct equivalent in Microsoft Excel.

### Google Sheets
- **Availability:** Full support in all versions since ~2018
- **Multiple Ranges:** Supports multiple range arguments
- **Empty Cell Handling:** Preserves empty cells in output
- **Array Integration:** Works seamlessly with other array functions

### Microsoft Excel
- **Native FLATTEN:** No function named FLATTEN exists
- **Alternatives:**
  - **TOCOL():** Excel 365 function that converts array to column (introduced 2022)
  - **TOROW():** Excel 365 function that converts array to row
  - **Power Query:** Unpivot feature for similar transformations
  - **VBA:** Custom functions can replicate FLATTEN behavior
- **Syntax Differences:** TOCOL has different parameters including scan order options

### Key Differences Summary

| Feature | Google Sheets (FLATTEN) | Microsoft Excel |
|---------|------------------------|-----------------|
| Function Name | FLATTEN | TOCOL/TOROW (365 only) |
| Multiple Ranges | Supported | Requires nesting |
| Scan Order | Row-by-row only | Configurable (TOCOL) |
| Empty Handling | Preserves empties | Can ignore or include |
| Availability | All versions | 365 only |

## Tips and Best Practices

1. **Use FILTER to remove empty cells:** FLATTEN preserves empty cells, which may not be desired. Use `FILTER(FLATTEN(range), FLATTEN(range)<>"")` to get only non-empty values.

2. **Remember the reading order:** FLATTEN reads left-to-right, then top-to-bottom. If you need column-by-column order, TRANSPOSE your data first: `FLATTEN(TRANSPOSE(range))`.

3. **Combine with UNIQUE for master lists:** `UNIQUE(FLATTEN(A:C))` creates a deduplicated list of all values across multiple columns—perfect for dropdown validation sources or reference lists.

4. **Enable multi-column aggregations:** Functions like COUNTUNIQUE, MAX, MIN, and AVERAGE work with flattened data, enabling aggregations across multiple columns that would otherwise require complex formulas.

5. **Watch for performance with large ranges:** FLATTEN on large ranges (especially full columns like A:A) can be slow. Limit ranges to actual data: use `A2:A` or a specific range like `A2:A1000`.

6. **Use for QUERY preprocessing:** When you need to QUERY across multiple columns as if they were one, FLATTEN them first: `QUERY(FLATTEN(range), "SELECT...")`.

7. **Combine multiple ranges strategically:** When flattening multiple ranges, they are appended in order. Use this to prioritize data sources or create combined datasets.

8. **Consider ARRAYFORMULA alternatives:** For some transformations, ARRAYFORMULA with INDEX/SEQUENCE may be more efficient than FLATTEN for very large datasets.
